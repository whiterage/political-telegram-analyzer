# Telegram Emotion Analyzer

**Telegram Emotion Analyzer** is a Go project for collecting, analyzing, and exporting data about audience emotional response to Telegram posts.

Проект делается как прикладной инструмент для эмпирического исследования политической коммуникации в Telegram. Основной фокус сейчас на анализе тестовых постов: реакции, просмотры, комментарии, репосты, формат публикации и эмоциональные фреймы.

## Цель проекта

Цель проекта — собрать базовый аналитический пайплайн для Telegram-постов:

1. Подготовка входных постов.
2. Подсчет метрик вовлеченности.
3. Определение формата публикации.
4. Классификация эмоционального фрейма.
5. Сортировка по уровню эмоционального отклика.
6. Отбор топовых публикаций.
7. Экспорт результатов в CSV.

## Что уже реализовано

Сейчас в проекте есть:

- доменные модели постов, реакций и результатов анализа;
- расчет общего количества реакций;
- расчет ERR;
- определение формата поста по длине текста;
- анализ одного поста и массива постов;
- сортировка постов по ERR;
- отбор Top-N публикаций;
- интерфейс классификатора эмоций;
- mock-классификатор для проверки архитектуры;
- экспорт результатов анализа в CSV.

Дополнительно в структуре уже заведены пакеты `keyword`, `manual` и `llm`, но они пока остаются заготовками под следующие этапы.

## Формула ERR

В проекте используется метрика **ERR** (Engagement Rate by Reach):

```text
ERR = (reactions + comments + forwards) / views * 100
```

Эта метрика позволяет сравнивать посты по вовлеченности, а не только по абсолютному числу реакций.

## Определение формата поста

Формат публикации определяется по длине текста:

```text
short  - до 300 символов включительно
medium - от 301 до 1000 символов
long   - больше 1000 символов
```

Сейчас приложение работает на тестовых данных и использует `mock`-классификатор эмоций.

## Запуск проекта

Из корня репозитория:

```bash
go run ./src/cmd/analyzer
```

После запуска приложение:

- проанализирует тестовые посты;
- выведет краткий результат в консоль;
- создаст CSV-файл `out_posts.csv` в корне проекта.

## Пример CSV

Файл экспорта содержит поля:

```text
id,channel_name,text,views,total_reactions,comments,forwards,err,format,emotion,frame,confidence,method,reason,markers
```

## Текущий статус

Проект находится на стадии MVP. Базовая аналитическая логика и CSV-экспорт уже работают, а подключение реальных источников данных и полноценная классификация эмоций пока в разработке.
